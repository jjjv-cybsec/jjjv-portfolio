# Entry 002 — Analyze your first packet
**Date:** 2025-04-01  
**Activity:** Analyze your first packet (Wireshark / pcap analysis)

## Description
I analyzed a packet capture (pcap) file to investigate traffic generated by a user connecting to an internet site. The exercise focused on using **Wireshark** to open the sample pcap, apply filters, inspect DNS and TCP flows, and extract observable details for documentation.

---

## Tools used
- **Wireshark**

---

## Executive summary / Key findings
- Observed DNS queries for `opensource.google.com` and multiple A records (e.g., `142.250.1.139`).  
- HTTP traffic generated by `curl` (User-Agent: `curl/7.74.0`) to port 80.  
- Local addresses observed: `172.21.224.2` (MAC `42:01:ac:15:e0:02`) and link-local `169.254.169.254` (MAC `42:01:ac:15:e0:01`).  
- No malicious indicators were identified in this lab capture; activity is consistent with normal DNS resolution and HTTP GET requests.

---

## Tasks mapped to the 5 W's (task-centric view)

### Task 1 — Explore data with Wireshark
**What (task):** Open `sample.pcap` and review high-level packet list.  
**Who (observables):** Source IP observed: `172.21.224.2` (MAC `42:01:ac:15:e0:02`).  
**When:** Packet timestamps appear in the capture (examples below).  
**Where:** Local VM network interface; traffic to external IPs.  
**Why (purpose):** Establish baseline and identify protocols present (ICMP, DNS, TCP).

### Task 2 — Apply a basic filter and inspect a packet
**What (task):** Apply `ip.addr == 142.250.1.139` and inspect TCP handshake and flags.  
**Who (observables):** Remote endpoint `142.250.1.139` (opensource.google.com).  
**When:** Example SYN packet observed in capture.  
**Where:** Source port `49652` → dest port `80`.  
**Why (purpose):** Verify connection establishment and capture TCP options (MSS, window scale, timestamps).

### Task 3 — Use filters to select packets by src/dst and MAC
**What (task):** Filters used: `ip.src == 142.250.1.139`, `ip.dst == 142.250.1.139`, `eth.addr == 42:01:ac:15:e0:02`.  
**Who:** Local host and remote host pairings identified.  
**When / Where / Why:** Narrow traffic to relevant flows for deeper inspection.

### Task 4 — Explore DNS packets
**What (task):** Filter `udp.port == 53` to inspect DNS queries/responses.  
**Key observation:** `Standard query 0x0c26 A opensource.google.com` and responses with multiple A records (`142.250.1.139`, `142.250.1.138`, `142.250.1.102`, ...).  
**Why:** Confirm name resolution and map domain to IPs used in subsequent HTTP requests.

### Task 5 — Inspect TCP/HTTP packets containing curl
**What (task):** Filter `tcp.port == 80` and `tcp contains "curl"`.  
**Key observation:** HTTP GET `/` with header `User-Agent: curl/7.74.0`.  
**Why:** Demonstrates how the pcap was generated (curl) and confirms HTTP-level content.

## Description
I am a security analyst investigating traffic to a website. I analyzed a packet capture (pcap) file that contains traffic data related to a user connecting to an internet site. This activity used **Wireshark** to open the sample pcap and apply filters to inspect network traffic.

## Tools used
**Wireshark**

## The 5 W's

**Who caused the incident**  
- Observed source addresses in the capture: `172.21.224.2` (MAC `42:01:ac:15:e0:02`) and `169.254.169.254` (MAC `42:01:ac:15:e0:01`) in some packets.

**What happened**  
- **Task 1:** Explore data with Wireshark (file `sample.pcap`). Observed ICMP (ping) and TCP flows.  
- **Task 2:** Applied filter `ip.addr == 142.250.1.139` and inspected TCP handshake details (example: `49652 → 80 [SYN] Seq=0 Win=65320 Len=0 MSS=1420 SACK_PERM ...`).  
- **Task 3:** Used filters `ip.src == 142.250.1.139`, `ip.dst == 142.250.1.139`, and `eth.addr == 42:01:ac:15:e0:02`.  
- **Task 4:** DNS inspection with `udp.port == 53` showed queries for `opensource.google.com` and responses mapping to addresses such as `142.250.1.139`.  
- **Task 5:** TCP inspection with `tcp.port == 80` and `tcp contains "curl"` revealed HTTP GET requests with `User-Agent: curl/7.74.0`.

**When did the incident occur**  
- Packet timestamps in the capture include entries on **Nov 23, 2022** (examples in the pcap).
